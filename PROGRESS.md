## Key Learnings

1. **Circomlibjs vs Circom Poseidon mismatch:** circomlibjs uses different Poseidon parameters than Circom's circomlib. For correct Merkle tree computation, use Circom's compiled WASM witness calculator.
2. **G2 point serialization:** snarkjs pi_b coordinates require careful ordering for arkworks/Soroban Bn254G2Affine: [x.c0, x.c1, y.c0, y.c1] where x.c0 = pi_b[0][1], x.c1 = pi_b[0][0], y.c0 = pi_b[1][1], y.c1 = pi_b[1][0].
3. **Soroban SDK versioning:** `crypto::bn254` module requires SDK 26+; `wasm32v1-none` target is required for Rust 1.82+.
4. **Public input order:** snarkjs public signals order follows witness signal declaration order (amount, root, nullifier), not Circom's `{public [...]}` declaration order.
